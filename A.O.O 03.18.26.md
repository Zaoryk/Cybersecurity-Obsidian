# RESUMEN — SEMANA 2: Arquitectura On-Premise vs Cloud

## 1. Conceptos Generales

### On-Premise
- Infraestructura local dentro de la empresa
- Control total de servidores y datos
- Alta inversión inicial (CAPEX)
- Escalabilidad limitada
- Mayor responsabilidad en seguridad y mantenimiento

### Cloud Computing
- Recursos TI a través de internet
- Pago por uso (OPEX)
- Escalabilidad dinámica
- Alta disponibilidad
- Gestión delegada al proveedor

---

## 2. Modelos de Servicio en la Nube

### IaaS (Infrastructure as a Service)
- Provisión de servidores, redes y almacenamiento
- Usuario gestiona SO y aplicaciones
- Ejemplo: máquinas virtuales

### PaaS (Platform as a Service)
- Entorno listo para desarrollar y desplegar aplicaciones
- El proveedor gestiona infraestructura
- Usuario se enfoca en el desarrollo

### SaaS (Software as a Service)
- Aplicaciones listas para usar vía web
- No requiere instalación local
- Ejemplo: CRM, ERP, correo

### BaaS (Backend as a Service)
- Servicios backend listos
- APIs, autenticación, bases de datos
- Reduce desarrollo desde cero

---

## 3. Pilares de la Arquitectura TI

### Networking
- Conectividad entre usuarios y sistemas
- Base de la comunicación

### Storage
- Almacenamiento de datos
- Persistencia de información

### Bases de Datos
- Organización y gestión de datos
- Soporte a operaciones críticas

### Aplicaciones
- Interfaz y lógica de negocio
- Interacción con el usuario

---

## 4. Comparación On-Premise vs Cloud

### On-Premise
- control total
- costos altos iniciales
- mantenimiento interno
- menor flexibilidad

### Cloud
- escalable
- flexible
- menor inversión inicial
- dependencia del proveedor

---

## 5. Arquitectura Moderna (Caso tipo Netflix)

### Uso de Cloud
- IaaS: infraestructura escalable
- PaaS: desarrollo ágil
- SaaS: plataforma de usuario
- BaaS: servicios backend

### Claves
- escalabilidad masiva
- tolerancia a fallos
- distribución global
- automatización

---

## 6. Problemas Comunes en Empresas (Caso EPTA)

### Conectividad
- redes inestables
- mala cobertura WiFi

### Comunicación
- canales no integrados
- pérdida de información

### Seguridad
- vulnerabilidades en IoT
- falta de monitoreo

### Innovación
- baja adopción tecnológica

---

## 7. Soluciones Propuestas

### Red
- mejora de infraestructura LAN/WiFi
- segmentación de red

### Conectividad
- implementación de SD-WAN
- VPN moderna

### Sistemas
- CRM omnicanal
- integración de plataformas

### Seguridad
- enfoque Zero Trust
- monitoreo continuo

### Innovación
- IoT
- IA para mantenimiento predictivo
- analítica de datos

---

## 8. Migración a la Nube

### Etapas
1. evaluación
2. migración parcial
3. optimización
4. transformación digital

---

## 9. Beneficios de la Nube

- escalabilidad
- disponibilidad
- eficiencia operativa
- reducción de costos a largo plazo
- innovación continua

---

## 10. Idea Clave

La nube no solo cambia la infraestructura, cambia el modelo operativo:

- de sistemas rígidos → a sistemas flexibles
- de gestión manual → a automatización
- de reacción → a predicción
