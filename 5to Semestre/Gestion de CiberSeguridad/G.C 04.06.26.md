# Metodologías de Gestión de Proyectos en Ciberseguridad

## 1. Concepto General

La gestión de proyectos en ciberseguridad consiste en planificar, ejecutar y controlar iniciativas destinadas a:

- proteger activos digitales
- reducir riesgos
- garantizar continuidad operativa
- cumplir normativas

Se diferencia de otros proyectos porque:
- trabaja con riesgos dinámicos
- requiere adaptación constante
- tiene alto impacto organizacional

## 2. Características de Proyectos de Ciberseguridad

| Característica | Descripción |
|---------------|------------|
| Alta incertidumbre | Amenazas cambian constantemente |
| Enfoque en riesgo | Se prioriza según impacto/probabilidad |
| Interdisciplinario | TI + negocio + legal |
| Crítico | Impacta continuidad del negocio |

## 3. Ciclo de Vida del Proyecto

| Fase | Descripción |
|------|------------|
| Inicio | Definir problema, objetivos, alcance |
| Planificación | Recursos, tiempos, riesgos |
| Ejecución | Implementación de soluciones |
| Monitoreo | Seguimiento y control |
| Cierre | Evaluación y documentación |

## 4. Metodologías Tradicionales

## Cascada (Waterfall)

### Características
- secuencial
- cada fase depende de la anterior
- documentación extensa

### Ventajas
- estructura clara
- fácil de gestionar

### Desventajas
- poca flexibilidad
- no se adapta a cambios rápidos

## 5. Metodologías Ágiles

## Agile

### Características
- iterativo e incremental
- entregas rápidas
- adaptación continua

### Ventajas
- flexibilidad
- respuesta rápida a amenazas

### Desventajas
- menor control documental
- requiere equipos maduros

## Scrum

### Componentes

| Elemento | Función |
|----------|--------|
| Sprint | ciclo corto de trabajo |
| Product Backlog | lista de tareas |
| Scrum Master | facilita proceso |
| Product Owner | define prioridades |

## Kanban

### Características
- flujo continuo
- visualización del trabajo
- mejora continua

## 6. Metodologías en Ciberseguridad

## NIST Framework

### Funciones

| Función | Descripción |
|--------|------------|
| Identify | identificar activos y riesgos |
| Protect | proteger sistemas |
| Detect | detectar incidentes |
| Respond | responder ataques |
| Recover | recuperar operaciones |

## ISO 27001

### Enfoque
- gestión de seguridad de la información

### Incluye
- políticas
- controles
- auditorías

## 7. Gestión de Riesgos

### Proceso

| Etapa | Descripción |
|------|------------|
| Identificación | detectar amenazas |
| Análisis | evaluar impacto |
| Evaluación | priorizar riesgos |
| Tratamiento | mitigar o aceptar |

## 8. Roles en Proyectos

| Rol | Función |
|-----|--------|
| Project Manager | coordina proyecto |
| Equipo TI | implementa soluciones |
| Seguridad | analiza riesgos |
| Stakeholders | interesados |

## 9. Factores Críticos de Éxito

- buena planificación
- gestión de riesgos
- comunicación efectiva
- recursos adecuados
- apoyo organizacional

## 10. Problemas Comunes

| Problema | Causa |
|----------|------|
| retrasos | mala planificación |
| sobrecostos | mala estimación |
| fallas de seguridad | mala implementación |
| resistencia al cambio | cultura organizacional |

## 11. Integración con Negocio

La ciberseguridad debe alinearse con:

- objetivos estratégicos
- continuidad operacional
- cumplimiento legal

## 12. Mejores Prácticas

- aplicar enfoque por riesgos
- usar metodologías ágiles en entornos dinámicos
- monitorear continuamente
- documentar procesos
- capacitar equipos
