# Arquitectura en la Nube vs On-Premise

## Concepto General
La arquitectura en la nube permite ejecutar aplicaciones y servicios usando infraestructura distribuida en internet en lugar de infraestructura local.

Ejemplo: Netflix transmite contenido a millones de personas simultáneamente utilizando infraestructura cloud escalable.

---

# Pilares Fundamentales de TI
Todas las aplicaciones modernas se basan en cuatro pilares tecnológicos.

## 1. Networking
- Sistema de conexión entre usuarios y servidores.
- Funciona como una **autopista digital** que transporta datos.

## 2. Storage
- Biblioteca digital donde se almacena información.
- Guarda archivos, contenido multimedia y datos de usuarios.

## 3. Bases de Datos
- Gestionan la información estructurada del usuario.
- Ejemplos:
  - Perfil de usuario
  - Historial de reproducción
  - Recomendaciones personalizadas

## 4. Aplicaciones
- Software o interfaz que utiliza el usuario.
- Ejemplo: aplicación de Netflix en celular, web o Smart TV.

---

# Infraestructura Tradicional (On-Premise)

## On-Premise
Infraestructura tecnológica ubicada físicamente dentro de la empresa.

Características:
- Servidores propios
- Mantenimiento interno
- Alto costo inicial
- Escalabilidad limitada

---

# Revolución de la Nube

La computación en la nube permite consumir infraestructura y servicios tecnológicos a través de internet.

Ventajas:
- Escalabilidad
- Flexibilidad
- Pago por uso
- Alta disponibilidad

## Servicios Informáticos Esenciales

### Networking (Redes)
- Conecta sistemas y dispositivos para compartir información de forma segura y eficiente.
- Funciona como la infraestructura de comunicación entre usuarios y servidores.

Ejemplo:
- VPN empresarial para que empleados accedan de forma segura a la red de la empresa.

---

### Bases de Datos
- Sistemas organizados para almacenar y gestionar datos electrónicamente.
- Permiten consultas rápidas y administración estructurada de la información.

Ejemplo:
- Sistema de inventario de retail que registra productos, precios y ventas en tiempo real.

---

### Storage (Almacenamiento)
- Hardware y software encargado de guardar datos digitales.
- Permite que la información esté disponible cuando se necesite.

Ejemplo:
- Servidor NAS utilizado para respaldos y copias de seguridad (backup).

---

### Aplicaciones
- Programas diseñados para realizar tareas específicas para usuarios o empresas.

Ejemplo:
- Sistema ERP utilizado por contabilidad para facturación, gestión de nómina y control financiero.

---

# Modelos de Servicio Cloud (XaaS)

## IaaS — Infrastructure as a Service
Entrega infraestructura virtualizada en la nube.

Incluye:
- servidores virtuales
- redes
- almacenamiento

Responsabilidades:
- Proveedor: infraestructura física
- Usuario: sistema operativo, middleware, datos y aplicaciones

Ejemplo:
- Servidor web en AWS usando una máquina virtual.

---

## PaaS — Platform as a Service
Proporciona una plataforma para desarrollar y desplegar aplicaciones en la nube.

Características:
- el proveedor gestiona infraestructura y entorno de ejecución
- el desarrollador se enfoca en el código

Ejemplo:
- Google App Engine para ejecutar aplicaciones sin administrar servidores.

---

## SaaS — Software as a Service
Aplicaciones completas accesibles a través de internet.

Características:
- uso mediante navegador o aplicación
- modelo de suscripción
- el proveedor gestiona toda la infraestructura

Ejemplos:
- Microsoft 365
- Zoom

---

## BaaS — Backend as a Service
Servicios backend listos para integrar en aplicaciones.

Incluye:
- autenticación de usuarios
- bases de datos
- notificaciones
- APIs

Ejemplo:
- Firebase de Google para gestionar usuarios y datos sin crear un servidor propio.

---

- escalar recursos según demanda
- reducir infraestructura física
- desarrollar aplicaciones más rápido
- ofrecer servicios globales a gran escala

## Concepto General
La arquitectura cloud permite ejecutar aplicaciones y servicios utilizando infraestructura distribuida en internet en lugar de servidores locales.

Permite que plataformas digitales escalen para atender millones de usuarios simultáneamente.

Ejemplo:
- Plataformas de streaming como Netflix utilizan infraestructura cloud para distribuir contenido globalmente.

# Conceptos de Redes Relacionados

Las plataformas cloud dependen de redes de datos para funcionar.

## Tipos de Redes
Las redes se clasifican según su tamaño y alcance.

PAN (Personal Area Network)
- alcance aproximado: hasta 10 metros
- ejemplo: Bluetooth

LAN (Local Area Network)
- alcance: edificios o campus
- ejemplo: red Ethernet de una empresa

MAN (Metropolitan Area Network)
- cubre ciudades completas

WAN (Wide Area Network)
- conecta redes a grandes distancias

# Idea Clave de la Computación en la Nube

La nube transforma la forma en que se construyen sistemas tecnológicos.

Ventajas principales:

Escalabilidad
- recursos aumentan o disminuyen según la demanda.

Confiabilidad
- redundancia y alta disponibilidad de servicios.

Innovación
- permite desarrollar aplicaciones rápidamente sin gestionar infraestructura física.
